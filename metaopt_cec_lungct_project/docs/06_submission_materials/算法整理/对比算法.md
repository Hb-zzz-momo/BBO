### 目标

给 **ROUTE_A_BUDGET_ADAPTIVE_BBO** 配一套**能直接落论文、也能落当前 MATLAB 基准框架**的对比算法名单。
我这里按你当前主线来定：**CEC（Congress on Evolutionary Computation，进化计算大会）基准测试是主验证路径，肺 CT（Computed Tomography，计算机断层扫描）只是应用验证；对比算法优先考虑 GWO（Grey Wolf Optimizer，灰狼优化）、PSO（Particle Swarm Optimization，粒子群优化）、DE（Differential Evolution，差分进化）这类仓库已明确支持或优先考虑的基线。**

### 每个算法为什么要进名单

#### 1. BBO_ORIG

这是**绝对不能缺**的。

原因很简单：

- 你改的就是 BBO（Beaver Behavior Optimizer，河狸行为优化）
- 不和原始 BBO 比，评审会直接问：
  **“那到底是 BBO 强，还是你的改进强？”**

这个算法的作用不是“充数量”，而是提供**最核心的因果证据**。

------

#### 2. PSO

PSO 是**最经典、最容易被评审认可**的群智能基线之一。
而且你的仓库说明里，**GWO / PSO / DE** 就是默认的 comparison baselines（对比基线）示例。

它的价值在于：

- 大家都熟
- 不是冷门算法
- 结果好坏很直观
- 很适合做“经典方法锚点”

------

#### 3. DE

DE 是**经典进化算法基线**，非常重要。

因为你的 `budget_adaptive`（预算自适应）本质上和“搜索资源如何分配”有关，
而 DE 代表的是另一条经典优化流派。
把它加进来，可以证明你的方法不是只在群智能里能赢，而是对传统强进化方法也有竞争力。

------

#### 4. GWO

GWO 是经典群智能里**最常见的强基线之一**。
仓库文档里也明确把它列为对比算法接入对象。

它的意义在于：

- 和 PSO 一起构成“经典群智能双锚点”
- 在很多 CEC 论文里都很常见
- 评审一看就知道你有没有过最基本的门槛

------

#### 5. WOA

WOA 适合放进来当一个**常见行为启发式基线**。

原因：

- 它在 BBO 原文对比集中就出现了
- 它和 BBO/HHO/GWO 这种群智能行为类算法风格接近
- 适合看“你的方法是不是只对某类算法占优，还是在同类行为启发式中也有竞争力”

------

#### 6. HHO

虽然你仓库现在**不再以 HHO 为中心**，但 HHO 仍然是很有代表性的对手。仓库里也明确说了“现在不以 HHO 为中心”，这恰恰意味着它适合作为**外部对手**，而不是研究主线本身。

把 HHO 放进来有两个好处：

- 它名气够大，评审熟
- 你前期又确实和 HHO 路线有历史关联，放进来很自然

------

#### 7. RIME

RIME 很值得放进**正式正文主表**。

原因：

- BBO 原文把它作为对手之一
- SBO 论文也把它列为“近期有代表性的强算法”之一
- 它不是纯经典老方法，而是能代表“近年强算法”

换句话说，RIME 能帮你回答：

> 我的改进不是只比老算法强，对近年的强方法也有竞争力。

------

#### 8. NRBO

NRBO 适合放在 **P2（增强层）**。

原因：

- 它也出现在 BBO 原文对比集中
- 属于“近年新强算法”类型
- 能增强论文说服力

但为什么不是 P1？
因为相比 PSO/DE/GWO/HHO/RIME，它的“公共认知度”稍弱一点，所以我把它放到增强层。

------

#### 9. SBO

这个是我**很建议你加，但不一定第一批就跑**的。

原因不是“同门算法所以要比”，而是：

- 你的仓库主线本来就同时围绕 **SBO 基线** 和 **BBO 基线**展开
- SBO 论文的结构和你很接近：
  **CEC 全局优化验证 + 统计检验 + 图像分割应用验证**

所以 SBO 是一个很好的**近邻研究对手**。
它特别适合回答：

> 不是随便找几个算法比，而是和“研究风格最接近”的方法也比过了。

------

#### 10. SHADE

SHADE 是我特意给你补进去的**强进化算法压轴基线**。

SBO 论文在对比算法选择时，特意纳入了 **SHADE、CMAES（Covariance Matrix Adaptation Evolution Strategy，协方差矩阵自适应进化策略）、LSHADE_cnEpSin** 等强进化算法，目的就是让对比集不只停留在“经典算法 + 行为类算法”，而是覆盖真正有战斗力的进化流派。

我这里选 **SHADE** 而不是一上来选 **CMAES** 或 **LSHADE_cnEpSin**，原因是：

- 代表性够强
- 进化算法属性明确
- 相对更容易解释其存在价值
- 比直接上太复杂的强基线更适合你当前阶段

------

### 为什么最终是这 10 个，而不是更多

我不是按“越多越好”选的，而是按这 5 个角色来配的：

| 角色                | 对应算法          |
| ------------------- | ----------------- |
| 母算法直接基线      | BBO_ORIG          |
| 经典群智能锚点      | PSO、GWO          |
| 经典进化锚点        | DE                |
| 常见行为/群智能近邻 | WOA、HHO          |
| 近年强对手          | RIME、NRBO、SHADE |
| 同研究风格近邻对手  | SBO               |

这样配有几个好处：

1. **不空泛**：不是一堆名字堆起来
2. **不单一**：不是全是一个流派
3. **不太轻**：不会显得对比集太弱
4. **不太重**：不会一上来把实验工程拖死

### 实验优先级怎么排

#### P0：必须先跑的最小闭环

先跑这 5 个：

- ROUTE_A_BUDGET_ADAPTIVE_BBO
- BBO_ORIG
- PSO
- DE
- GWO

用途：

- 验证主框架是否稳定
- 先确认你相对母算法和经典基线有没有基本优势
- 先出第一版 summary（汇总表）、mean（均值）、std（标准差）、收敛曲线

这一步最重要，因为它能最快告诉你：

> 这条主线值不值得继续大铺实验面

------

#### P1：正文核心对比层

在 P0 基础上加：

- WOA
- HHO
- RIME

此时你就有一套**很像论文正文主表**的对比集了。
如果这一层结果已经比较稳，其实正文主实验已经够用了。

------

#### P2：增强说服力层

最后再加：

- NRBO
- SBO
- SHADE

这层的作用不是“救主线”，而是：

- 增强论文厚度
- 提高对比覆盖面
- 防止评审说“只比了常见算法，没比真正强的”

------

### 公平性设置建议

这部分很关键，不然算法选得再好也会被质疑。

SBO 论文明确写了：对比时采用**相同种群规模、相同最大函数评估次数、相同终止条件**，参数优先沿用原论文推荐配置。
你的仓库规则里也明确要求：**不要悄悄改 benchmark suite（基准套件）、dimension（维度）、population size（种群规模）、function evaluation budget（函数评估预算）、number of runs（重复次数）和 stopping criteria（停止条件）**。

我建议你统一成下面这套：

| 项目                        | 建议                                |
| --------------------------- | ----------------------------------- |
| Population size（种群规模） | 全算法统一                          |
| MaxFEs（最大函数评估次数）  | 全算法统一                          |
| Runs（独立重复次数）        | 全算法统一                          |
| Bounds（边界）              | 按 suite 原定义，不私改             |
| Stop rule（停止条件）       | 全算法一致                          |
| 参数设置                    | 优先原论文推荐值                    |
| 输出                        | raw results + summary + convergence |

如果你当前主线已经在跑 **30000 / 300000** 两档预算，那么：

- **300000**：作为正式主结果
- **30000**：作为低预算压力测试 / 补充实验

这样最合理。

### 我不建议你现在优先放进最终名单的算法

下面这些不是不能比，而是**当前阶段优先级不高**：

#### 1. TLBO（Teaching-Learning-Based Optimization，教学相长优化）

原因：人类行为启发式代表意义有，但和 SBO 角色有重合。

#### 2. PO（Political Optimization，政治优化）

原因：代表性可以，但没有 SBO 对你更贴近。

#### 3. CMAES

原因：很强，但实现/适配/解释成本偏高，适合后续增强版。

#### 4. LSHADE_cnEpSin

原因：很强，但现阶段过重，先上 SHADE 就够了。

### 论文里怎么写这套对比集

你后面实验章节可以直接按这个逻辑写：

#### 对比集构成逻辑

- 母算法基线：BBO_ORIG

- 也不是故意避开近年的强方法

### 论文撰写建议

这一段在论文里不要写成“为了全面比较，我们选择了若干算法”。
这种写法太空。

更好的写法是：

> 为了全面评估 ROUTE_A_BUDGET_ADAPTIVE_BBO 的优化性能，对比算法从四类代表性方法中选取：一类为母算法 BBO，用于验证改进模块的直接收益；一类为经典元启发式基线，包括 PSO、DE 和 GWO，用于提供公认的性能锚点；一类为常见高性能群智能与行为启发式算法，包括 WOA、HHO 和 RIME；另一类为近年具有较强竞争力的优化方法，包括 NRBO、SHADE 以及与本文研究风格相近的 SBO。该对比集兼顾经典性、代表性与先进性，能够较为全面地反映所提方法的实际竞争力。

这段你后面基本可以直接改一改就进论文。
