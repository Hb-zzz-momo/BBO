### 直接结论

我看了你上传的 **BBO 原始论文**《*Beaver behavior optimizer: A novel metaheuristic algorithm for solar PV parameter identification and engineering problems*》，它**不是只看 mean（均值）**，而是用了**一整套指标组合**。

------

### BBO 原始论文在基准测试里用了什么指标

在 **CEC2017** 和 **CEC2022** 基准实验部分，论文明确写了：

- **mean（均值）**
- **standard deviation / std（标准差）**
- **Wilcoxon signed-rank test（Wilcoxon 符号秩检验）**
- **Friedman mean rank test（Friedman 平均秩检验）**

而且这些结果来自**多次独立运行**；论文写的是 **30 次独立运行**，再统计均值和标准差。

------

### 具体怎么用的

#### 1. Mean（均值）

这是他们衡量“平均优化效果”的核心数值。

论文在 CEC2017/CEC2022 的主结果表里，都是按各函数给出：

- **Avg**
- **Std**

也就是：

- `Avg` 看平均表现
- `Std` 看稳定性

论文自己在结果分析里也反复强调：
BBO 在多数函数上取得了更小的 **mean** 和 **std**。

------

#### 2. Std（标准差）

这用来衡量算法的**鲁棒性（robustness，稳定程度）**。

论文在摘要和实验分析里都把“**convergence and robustness（收敛性和稳健性）**”作为优势表述；具体落地方式就是看 **mean + std**。

------

#### 3. Wilcoxon signed-rank test（显著性检验）

这个是拿来判断：

> **BBO 比对手好，是不是统计上真的有意义，而不是随机碰巧。**

论文写得很明确：

- 当 **p-value < 0.05** 时，认为与 BBO 有显著差异
- 用 **“+/=/−”** 表示 BBO 相对对手是更好、相近还是更差。

也就是说，原论文不是“我看均值更小就说赢了”，而是还补了**统计显著性证据**。

------

#### 4. Friedman mean rank test（平均排名检验）

这个指标回答的是：

> **在整套 benchmark（基准测试）上，BBO 的整体档次排第几。**

论文在 CEC2017 和 CEC2022 上都给了 **Friedman 排名表**，而且还配了 **radar chart（雷达图）** 去展示排名。

这说明原论文非常重视：

- 单函数数值
- 全体函数上的整体排序

------

### 除了表格指标，原论文还看了哪些“过程型证据”

原论文专门有一节叫 **Convergence behavior evaluation（收敛行为评估）**。这里它不只是看最终值，还看：

- **2D contour（二维等高线）**
- **最终搜索个体位置**
- **mean fitness value over iterations（迭代过程中的平均适应度变化）**
- **第一维搜索轨迹**
- **convergence curve（收敛曲线）**
- **boxplot（箱线图）**
- **radar chart（雷达图）**

论文明确写到，图中第三列画的是 **mean fitness value over iterations**，也就是“平均适应度随迭代变化”；后面在结果分析里还用了 **convergence curves** 和 **boxplots** 来支持“收敛快、波动小”的说法。

------

### 所以，BBO 原始论文的“主证据链”是什么

你可以把它总结成这一套：

#### 基准函数部分

1. **Avg（均值）**
2. **Std（标准差）**
3. **Wilcoxon 检验**
4. **Friedman 平均排名**
5. **Convergence curves（收敛曲线）**
6. **Boxplots（箱线图）**
7. **Radar charts（雷达图）**
8. **搜索轨迹/种群分布图**

也就是：

> **均值 + 稳定性 + 显著性 + 排名 + 动态过程可视化**

这才是它的完整评价框架。

------

### 在 PV 参数辨识和工程问题里，它又用了什么指标

这部分更直观。
论文后面的 **Single diode model（单二极管模型）**、**Double diode model（双二极管模型）**、**PV module model（光伏模块模型）**，以及几个工程设计问题表格，统一用了：

- **Max**
- **Min**
- **Mean**
- **Std**
- **Wilcoxon**
- **Friedman Value**

例如论文中的 Table 13、14、15、16、17、18、19 都是这个格式。

这说明在应用问题里，原论文甚至比 benchmark 部分更完整，因为它把：

- 最好情况（Min / Max，视问题而定）
- 平均水平（Mean）
- 稳定性（Std）
- 显著性（Wilcoxon）
- 总体排名（Friedman）

全都一起放进表里了。

------

### 一个很关键的结论

#### BBO 原始论文不是“只拿 mean 说话”

而是：

- **benchmark 上：Avg + Std + Wilcoxon + Friedman + 曲线/箱线图/雷达图**
- **应用上：Max + Min + Mean + Std + Wilcoxon + Friedman**

所以如果你问：

> “原始论文是不是只抓着 mean？”

答案是：

> **不是。mean 只是其中一项，而且不是唯一决定性证据。**

------

### 那对你现在的实验有什么启发

最重要的一点是：

#### 你现在如果想更贴近 BBO 原始论文的写法，至少要补三样

1. **std（标准差）**
2. **Wilcoxon / Friedman**
3. **收敛曲线 / 箱线图**

因为这三样正是原论文已经在做的。
只用 mean 去判断方向可以，但如果要往“论文级证据”靠，就不够了。

------

### 更进一步：原论文更像在回答什么问题

它的评价逻辑不是单纯问：

- “平均值小不小？”

而是在同时回答：

- **平均效果好不好**
- **波动大不大**
- **差异是不是显著**
- **整体名次高不高**
- **收敛过程漂不漂亮**

这就是为什么它更像一篇完整论文，而不是一张结果表。
