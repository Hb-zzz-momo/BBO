### 四、主循环核心：你这版 V3 的“大脑”在哪里

### 4.1 阶段因子 `E`

```matlab
progress = t / Max_iteration;
E = sin((pi / 2) * progress);
```

#### 作用

`E` 随迭代从 0 单调增加到 1。

#### 和原始 BBO 的关系

这其实就是原始 BBO 的 **Dam-Phase factor（坝体阶段因子）** 的直接继承，只是原文记作 `D`，不是 `E`。原论文中 `D = sin(πt/2T)`，并用一个随机数决定当前进入 exploration 还是 exploitation。

#### 直观理解

- 前期 `E` 小：更偏探索
- 后期 `E` 大：更偏开发

所以这个变量决定了整段算法的“节奏感”。

------

### 4.2 architects 的选取

```matlab
[~, sorted_idx] = sort(fitness);
architects_idx = sorted_idx(1:max(2, round(N * 0.25)));
```

#### 作用

把当前适应度最好的前 25% 个体选为 architects。

#### 和原始 BBO 的关系

这正对应原始 BBO 里“architects are the top 25% of the population”的设定。

#### 你要真正理解的点

这里的 architect **不是单纯的“局部搜索个体”**。
在原始 BBO 里，architect/prospector 是 **探索阶段内部的两类角色分工**：

- **architect**：更像“优质样本之间的互相借鉴”
- **prospector**：一边向 architect 学，一边保留随机探测能力

所以 architect 不是“纯局部”，而是 **探索阶段里偏收缩、偏有方向性的那一支**。

## 五、逐段拆解主更新公式

------

### 5.1 `if rand < E` 分支：开发阶段主更新

```matlab
if rand < E
    for j = 1:dim
        k = randi([1, N]);
        while k == i
            k = randi([1, N]);
        end
        population(i, j) = population(i, j) + rand * (population(k, j) - population(i, j)) ...
            + rand * (best_solution(j) - population(i, j));
    end
```

#### 这段在做什么

对第 `i` 个个体的第 `j` 维做更新：

- 第一项：朝随机个体 `k` 靠拢
- 第二项：朝当前全局最优 `best_solution` 靠拢

#### 数学意义

可以写成：

$ x_{ij}^{new}=x_{ij}+r_1(x_{kj}-x_{ij})+r_2(x_{best,j}-x_{ij})$

这本质上是一个 **双引导收缩**：

- `population(k,j) - population(i,j)`：保持群体信息交换
- `best_solution(j) - population(i,j)`：强化朝最优解方向收敛

#### 和原始 BBO 的关系

这基本对应原始 BBO 的 exploitation（开发）更新思想：
个体在开发阶段不再去找新材料，而是向其他成员和最优个体学习，持续改进现有材料。原文第 4 页明确写到：开发阶段 “improve the existing materials by learning from the individual with the best combination of materials, as well as other colony members”。

#### 这段的能力定位

这段更偏：

- **后期收敛**
- **简单函数精修**
- **向最优区域快速压缩**

#### 风险

如果这段太强，会导致：

- 种群多样性（diversity，差异性）下降
- 多峰函数更容易早熟收敛（premature convergence，过早收敛）

------

### 5.2 `else` 分支第一部分：architect 的探索式学习

```matlab
else
    if ismember(i, architects_idx)
        for j = 1:dim
            if rand < 0.5
                k = architects_idx(randi(length(architects_idx)));
                population(i, j) = population(i, j) + rand * (population(k, j) - population(i, j));
            end
        end
```

#### 这段在做什么

如果当前个体是 architect：

- 每一维有 50% 概率
- 向另一个 architect 学习

#### 和原始 BBO 的关系

这正对应原始 BBO 探索阶段中 architect 的机制：
**architect 以 50% 概率向另一个 architect 学习。**

#### 直观理解

这不是乱跑，而是：

> **在优秀群体内部做有约束的横向借鉴**

也就是说它虽然还处在探索阶段，但已经不是“完全随机探索”，而是：

- 在优质子群体内部传播好维度
- 让优秀个体之间互相“拼装材料”

#### 能力定位

这段更偏：

- **有方向的探索**
- **精英之间的信息扩散**
- **降低完全随机带来的噪声**

------

### 5.3 `else` 分支第二部分：prospector 的探索更新

```matlab
    else
        for j = 1:dim
            if rand < 0.5
                k = architects_idx(randi(length(architects_idx)));
                population(i, j) = population(i, j) + rand * (population(k, j) - population(i, j));
            else
                disturbance = cos((pi / 2) * progress) * (ub(j) - lb(j)) * randn / 10;
                population(i, j) = population(i, j) + disturbance;
            end
        end
    end
end
```

#### 这段在做什么

如果不是 architect（也就是 prospector）：

- 50% 概率向 architect 学习
- 50% 概率加一个高斯扰动（Gaussian disturbance，高斯扰动）

#### 和原始 BBO 的关系

这几乎就是原始 BBO 探索阶段 prospector 的核心：
prospector 一方面向 architect 学，另一方面保留随机材料探索；而且扰动项会随 `cos(πt/2T)` 逐渐减小。

#### 扰动项为什么重要

```matlab
disturbance = cos((pi / 2) * progress) * (ub(j) - lb(j)) * randn / 10;
```

这里有三层含义：

1. `randn`：高斯随机扰动，能向左右两边探索
2. `(ub(j)-lb(j))`：让步长与搜索空间尺度匹配
3. `cos((pi/2)*progress)`：前期大，后期小

#### 所以它的节奏是

- **前期**：扰动大，鼓励广泛搜索
- **后期**：扰动缩小，避免破坏已找到的好区域

#### 这一段最关键的理解

这才是原始 BBO 里最“显性”的全局探索来源。
所以如果你问“architect 和 prospector 谁更全局”：

- **architect**：偏精英导向的有方向探索
- **prospector**：偏随机扩散的广域探索

## 七、适应度更新：这里有一个非常关键的问题

```matlab
new_fitness = fobj(population(i, :));
if new_fitness < fitness(i)
    fitness(i) = new_fitness;
    if new_fitness < best_fitness
        best_fitness = new_fitness;
        best_solution = population(i, :);
    end
end
```

### 这段“按意图”是在做什么

它想做的是 **greedy selection（贪婪选择）**：

- 新解更好才接受
- 顺便更新全局最优

### 但“按实际执行”有一个严重隐患

你前面已经把：

```matlab
population(i, j) = ...
```

直接写回了 `population(i,:)`。

也就是说：

- **位置 `population(i,:)` 已经变成新位置**
- 但如果 `new_fitness >= fitness(i)`，你**没有把旧位置还原**
- 同时 `fitness(i)` 却还保留旧适应度

这会导致：

> **population 和 fitness 不同步**

也就是：

- `population(i,:)` 是新位置
- `fitness(i)` 却是旧位置的适应度

### 这个 bug 会带来什么后果

后面你又用了：

```matlab
[~, sorted_idx] = sort(fitness);
elite_pool = population(elite_sorted(1:elite_pool_size), :);
```

这就意味着：

- 排序依据是旧适应度
- 但取出的 `population` 却可能已经不是对应那个适应度的位置了

所以后面的：

- architects 选择
- elite_pool 构造
- 差分局部搜索

都有可能建立在 **错配的“位置-适应度关系”** 上。

### 这个问题很关键

这不是“小瑕疵”，而是会直接影响实验可信度的问题。

------

### 你应该怎么理解它

如果按研究角度说：

- **V3 的设计思路是合理的**
- **但当前实现里，这里存在位置-适应度不同步 bug**

这很可能会让你看到某些结果“看起来有效，但不稳定”。

## 八、V3 真正新增的改进点：精英差分局部搜索

```matlab
elite_pool_size = max(4, round(0.2 * N));
[~, elite_sorted] = sort(fitness);
elite_pool = population(elite_sorted(1:elite_pool_size), :);

if size(elite_pool, 1) >= 3
    ids = randperm(size(elite_pool, 1), 3);
    e1 = elite_pool(ids(1), :);
    e2 = elite_pool(ids(2), :);
    e3 = elite_pool(ids(3), :);

    F = 0.6 - 0.4 * progress;
    candidate = best_solution + F * (e1 - e2) + 0.1 * rand(1, dim) .* (e3 - best_solution);
    candidate = max(candidate, lb);
    candidate = min(candidate, ub);

    candidate_fitness = fobj(candidate);
    if candidate_fitness < best_fitness
        best_fitness = candidate_fitness;
        best_solution = candidate;
    end
end
```

------

### 8.1 这段是 V3 的核心创新

这段不是原始 BBO 自带的。
这是你这版 V3 真正额外添加的 **elite differential local search（精英差分局部搜索）**。

------

### 8.2 它在做什么

#### 第一步：构建精英池

```matlab
elite_pool_size = max(4, round(0.2 * N));
```

取前 20% 的个体作为精英池。

#### 第二步：随机抽 3 个精英

```matlab
e1, e2, e3
```

#### 第三步：围绕当前最优解造一个候选解

```matlab
candidate = best_solution + F * (e1 - e2) + 0.1 * rand .* (e3 - best_solution);
```

这实际上融合了两类信息：

- `F * (e1 - e2)`：差分方向，提供局部微调方向
- `0.1 * rand .* (e3 - best_solution)`：在最优附近再加一个轻量扰动

#### 第四步：只更新全局最优

它没有替换种群，只是拿来试图刷新 `best_solution`。

------

### 8.3 这段为什么有效

#### 原因 1：在精英子空间里找更优方向

差分向量 `(e1 - e2)` 来自优秀个体之间的差异，通常比完全随机扰动更有信息量。

#### 原因 2：围绕 `best_solution` 做局部精修

因为候选解以 `best_solution` 为中心，所以它天然偏 exploitation（开发）。

#### 原因 3：`F` 随迭代减小

```matlab
F = 0.6 - 0.4 * progress;
```

- 前期 `F` 较大：搜索半径稍大
- 后期 `F` 较小：搜索更细

所以它天然适合：

- 简单函数末期逼近最优
- 单峰函数精度提升
- 原始 BBO “已经接近，但最后一截不够细”的场景
