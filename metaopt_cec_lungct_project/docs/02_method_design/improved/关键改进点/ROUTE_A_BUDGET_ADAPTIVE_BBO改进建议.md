这说明：

> 它平时能把问题压到一个“还可以”的层次，

所以 F10 的问题不像 F11 那么“系统性错盆地”，而更像：

- 但**鲁棒性（robustness，稳定抵抗坏情况的能力）还不够**。

#### 3）F12：能慢慢改，但后劲明显不如强算法

- `ROUTE_A_BUDGET_ADAPTIVE_BBO`：**2861.996 ± 2.096**

你这版不是完全不动，而是：

这说明：

> route 版的后期搜索寿命太短。

- **机制没触发**

### 3. 其他强算法到底强在哪里

这里我不建议简单说“哪个算法强就抄哪个算法”。

更有价值的是：

> **看它们为什么能解决你现在这个特定痛点。**

#### SBO（Status-based Optimization，状态优化）

SBO 的核心思想是：个体会靠近“高地位圈层”，并向高状态同伴学习，同时强调 exploration（探索）和 exploitation（开发）的平衡。([ScienceDirect](https://www.sciencedirect.com/science/article/abs/pii/S0925231225012755))

结合你现在的结果，我觉得 SBO 真正值得借鉴的不是“社会学隐喻”，而是这两点：

1. **不是只盯全局最优一个点，而是在“高质量圈层”里学**
2. **个体更新带有等级关系 / 圈层关系，不是单点吸附**

这很适合修你现在 F11 的问题，因为你当前太像：

- 大家一起朝一个方向收缩；
- 一旦这个方向错了，全队都错。

而 SBO 的“圈层学习”更像：

- 不是只看第一名；
- 而是在前排优质个体群里做结构化学习。

#### PLO（Polar Lights Optimizer，极光优化器）

PLO 的论文强调其灵感来自带磁场约束的高能粒子运动轨迹，核心不是简单随机跳，而是**带方向曲率的连续迁移**，并且作者在 CEC2014 和 CEC2022 上做了系统评估。([ScienceDirect](https://www.sciencedirect.com/science/article/abs/pii/S0925231224011986))

结合你结果看，PLO 值得借鉴的是：

1. **轨迹不是“直缩”，而是“弯着走”**
2. **后期搜索不会很早死掉**
3. **能在很后面还继续翻盆地**

这和你 F11 / F12 的问题非常对口。

你的算法现在更像“直线冲刺后原地蹲守”，PLO 则像“沿着曲线慢慢绕进去，最后找到正确入口”。

#### DE 系列（Differential Evolution，差分进化）里那条真正值得借的主线

这里我反而更想提醒一句：

> 真正值得借鉴的，不一定是最新换皮名字，
> 很多最耐用的机制，其实来自 **JADE / SHADE / L-SHADE** 这条线。

- **JADE** 提出了 `current-to-pbest` 和 **external archive（外部档案）**；([ahmetcevahircinar.com.tr](https://ahmetcevahircinar.com.tr/wp-content/uploads/2017/01/JADE_Adaptive_Differential_Evolution_with_Optional_External_Archive.pdf))
- **SHADE** 用 **success-history（成功历史）** 去自适应控制参数；
- **L-SHADE** 在此基础上加了 **linear population size reduction（线性种群缩减）**，并且论文报告它对混合函数很有竞争力；不过作者也明确提醒：当 hybrid（混合）目标的组成部分特性差异非常大时，自适应 DE 的表现也会明显下降。([metahack.org](https://metahack.org/CEC2014-Tanabe-Fukunaga.pdf))

这和你现在的情况非常像：

- 当前 route 版对“普通可收缩区域”有效；
- 但对“特性差异大、层级切换强、盆地复杂”的函数容易失效。

所以：

> **别只盯“新算法名字”，要盯“机制”。**
> 你现在最缺的机制，DE 家族其实已经研究得很成熟了。

### 4. 近期论文里，哪些机制最值得借来修你的问题

我按“和你当前痛点的匹配度”来排。

#### 第一梯队：最值得借

| 借鉴机制                                         | 来源            | 为什么和你匹配                                               |
| ------------------------------------------------ | --------------- | ------------------------------------------------------------ |
| **success-history 参数记忆**                     | SHADE / L-SHADE | 解决“步长 / 扰动强度不是固定收缩”的问题                      |
| **external archive + replay（外部档案 + 回放）** | JADE            | 解决“进错盆地后，没有历史好方向可回用”                       |
| **stagnation-aware jump（停滞感知跳跃）**        | ADE-AESDE 2025  | 直接针对后期停滞、多样性骤降                                 |
| **dual dispersal（双重扩散）**                   | DDSBO 2026      | 直接针对 premature convergence（过早收敛）和 local optimum（局部最优） |
| **trend-aware direction（趋势感知方向）**        | TAM 2025        | 解决“只会缩，不会沿有效趋势修正”                             |

具体看文献：

- **ADE-AESDE（2025）** 提出：用 **adaptive stagnation index（自适应停滞指数）**、**population hypervolume（种群超体积）停滞检测**、**guided differential jump（引导差分跳跃）**、**seed-pool recombination（种子池重组）** 和 **archive-guided replay（档案引导回放）** 来处理后期多样性下降和停滞。这个和你现在的问题几乎是“同病同治”。([ScienceDirect](https://www.sciencedirect.com/science/article/pii/S2590123025026209))
- **DDSBO（2026, Neurocomputing）** 明确就是针对 SBO 的 premature convergence（过早收敛）问题，加入 **dual dispersal mechanisms（双重扩散机制）** 来提升多样性、平衡探索和开发、增强逃离局部最优能力。([ScienceDirect](https://www.sciencedirect.com/science/article/abs/pii/S0925231225030668))
- **Trend-Aware Mechanism（2025, Applied Soft Computing）** 明确提出：利用最近两次迭代的位置历史，构造 **trend line（趋势线）** 来增强更新方向。([ScienceDirect](https://www.sciencedirect.com/science/article/pii/S1568494625008166/pdf?utm_source=chatgpt.com))

#### 第二梯队：可借，但要克制

- **Comprehensive Learning（综合学习）**
  - CLMGO（2025）强调让个体向**多个高质量解**学习，而不是只盯单一最优；同时它专门把 MGO 中冗余机制删掉来提升效率。([ScienceDirect](https://www.sciencedirect.com/science/article/abs/pii/S0925231225012755?utm_source=chatgpt.com))
  - 这个适合你，但建议轻量引入，不要把主干改得太重。
- **Restart with growing population（逐步放大种群的重启）**
  - IPOP-CMA-ES（Increasing Population CMA-ES，增种群重启 CMA-ES）这类思路，本质是：**重启后让搜索更 global（全局）**。([CMap](https://www.cmap.polytechnique.fr/~nikolaus.hansen/cec2005ipopcmaes.pdf?utm_source=chatgpt.com))
  - 这个概念对你有启发，但不建议直接整套移植。

### 5. 我建议不要“换算法”，而是换主线机制

如果论文主线还是 **“改进 BBO”**，那最好的做法不是换成 SBO / PLO / DE。

而是：

> **把 BBO 从“预算自适应版”升级成“状态感知的记忆-扩散-回放版”。**

我给一个更适合你当前问题的下一版主线：

### 推荐主线：`State-Aware Archive-Dispersal BBO`

可以理解成 4 层：

#### 第 1 层：停滞检测层

不要只看 `gbest`（global best，全局最优）有没有变。

要同时看：

- `Δgbest`（全局最优改善幅度）
- population radius（种群半径）
- elite turnover（精英集合更替率）
- hypervolume / spread（群体分散度）

因为你现在最大的问题就是：

- 表面看“已经很稳”
- 实际上是“稳稳地卡死了”

#### 第 2 层：档案记忆层

至少保留两类 archive（档案）：

- **elite archive（精英档案）**：历史上真正好的解
- **diverse archive（多样档案）**：和当前盆地差异比较大的解

这样一来，后期停滞时不是“乱跳”，而是：

> **从历史上真正有效、又不完全重复当前盆地的信息里重组。**

#### 第 3 层：动作层

动作我建议不是一个，而是三个：

1. **status learning（状态圈层学习）**
   - 学前 p% 精英，不只学 gbest
2. **trend move（趋势引导移动）**
   - 用最近若干代 elite centroid（精英中心）形成趋势方向
3. **dual dispersal（双重扩散）**
   - 近距扩散：围绕精英圈微弯轨迹搜索
   - 远距扩散：对坏 run 的个体做 archive-guided differential replay

#### 第 4 层：参数记忆层

把你现在“预算自适应”的调节逻辑升级成：

- **成功历史驱动的步长 / 扰动强度记忆**
- 不要只按预算单调缩小

因为你现在最大的症状之一就是：

> 到了后期，参数像“只会变小，不会反弹”。

### 6. 如果只允许做一次大改，我会怎么选

如果现在只做一条真正有希望出效果的新主线，我建议：

### 方案：**Archive + Dual Dispersal + Success History**

别再优先做单纯 rescue 壳子。

直接做下面这个核心组合：

#### 必做 1：外部档案回放

给停滞个体一个“有记忆的差分重放”：

- 从 elite archive 取一个好点
- 从 diverse archive 取一个远点
- 做一次差分 / 重组

作用：

- 不再是瞎跳
- 而是“带历史方向感的重定位”

#### 必做 2：双重扩散

一近一远：

- 近扩散：修 F10 / F12 这种“差一点”的情况
- 远扩散：救 F11 这种“盆地错了”的情况

#### 必做 3：成功历史自适应

记录“哪些步长 / 强度 / 组合方式真的带来了 improvement（改进）”。

这样参数调节不是拍脑袋，而是：

- 哪种动作在当前函数类型上有效，
- 后面就多给它权重。

- 提出一种**趋势引导的双层重定位更新方法**

这样更像“自己的方法”，不是“别人模块拼盘”。
