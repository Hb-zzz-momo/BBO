# V3 Improved/Ablation 版本说明（含代码块）

本文档用于解释 `BBO_v3_*` 各版本相对 `V3` 主干到底改了什么，以及这些改进在代码中的对应位置。

## 1. 结构总览

- 统一核心：`BBO_improved_v3_ablation_core.m`
- 薄封装入口：`BBO_v3_*.m`
- 基线封装：`BBO_v3_baseline.m`（直接调用 `BBO_improved_v3`）

核心思想：

- A 类改进：简单函数收敛增强（`fast_simple_a` / `fast_simple_b`）
- B 类改进：条件触发 directional（`dir_*`）
- Hybrid：A + B 组合

更细一点理解整个工程结构：

- **统一核心文件**负责：
  - 种群初始化
  - 适应度（fitness，适应度值）计算
  - 主迭代循环
  - 各种改进模块的真实执行
  - 最优解更新
  - 收敛曲线记录
- **薄封装入口文件**负责：
  - 暴露统一接口
  - 把某个字符串 `mode` 传入核心函数
  - 让 benchmark（基准测试）框架把每个版本当成一套独立算法来调用
- **baseline 封装**的作用是：
  - 给消融实验一个“零改动锚点”
  - 保证 `V3` 主干和各种 ablation（消融）版本能放在统一框架里公平比较

从代码设计角度看，这种结构的优点是：

1. **接口统一**：所有版本都长得像
   `function [best_fitness, best_solution, Convergence_curve] = xxx(N, Max_iteration, lb, ub, dim, fobj)`
2. **最小侵入**：改进逻辑集中在一个核心文件里，不用复制很多份主循环
3. **便于消融**：只改 `mode` 就能开关某个模块
4. **便于论文写法**：可以明确区分
   - baseline
   - 单模块增强
   - 条件触发增强
   - 组合增强

## 2. 模式映射代码（所有版本的根）

以下代码定义了每个 `mode` 开启哪些模块：

```matlab
function cfg = mode_config(mode, max_iter)
    cfg = struct();

    cfg.use_fast_A = false;
    cfg.use_fast_B = false;
    cfg.dir_late = false;
    cfg.dir_stagnation = false;
    cfg.dir_elite_only = false;
    cfg.dir_small_step = false;

    cfg.stall_window = max(5, round(0.08 * max_iter));
    cfg.late_start = 0.60;

    switch char(mode)
        case 'baseline'
        case 'fast_simple_a'
            cfg.use_fast_A = true;
        case 'fast_simple_b'
            cfg.use_fast_B = true;
        case 'dir_late'
            cfg.dir_late = true;
        case 'dir_stagnation'
            cfg.dir_stagnation = true;
        case 'dir_elite_only'
            cfg.dir_elite_only = true;
        case 'dir_small_step'
            cfg.dir_small_step = true;
            cfg.dir_late = true;
        case 'hybrid_a_dir_stag'
            cfg.use_fast_A = true;
            cfg.dir_stagnation = true;
        case 'hybrid_b_dir_small'
            cfg.use_fast_B = true;
            cfg.dir_small_step = true;
            cfg.dir_late = true;
        otherwise
            error('Unsupported v3 ablation mode: %s', mode);
    end
end
```

### 2.1 这段函数在做什么

这段函数本质上是在做一件事：

> 把一个字符串形式的实验模式 `mode`，翻译成一个配置结构体 `cfg`，让主循环知道“这次该开哪些模块”。

也就是说，它不是直接优化，而是在做**策略装配（strategy assembly，策略组装）**。

------

### 2.2 逐行语法解析

#### `function cfg = mode_config(mode, max_iter)`

这是 MATLAB（矩阵实验室）函数定义语法：

- `cfg`：函数输出
- `mode`：输入参数，表示当前算法模式
- `max_iter`：最大迭代次数，用于计算一些相对阈值

可以理解成：

> 给我一个模式名和总迭代数，我返回一个配置表。

------

#### `cfg = struct();`

`struct()` 用来创建**结构体（structure，键值式数据容器）**。

这里创建的是一个空结构体，后面通过：

```matlab
cfg.use_fast_A = false;
```

这种写法，动态添加字段。

这比写很多独立变量更适合做“模块开关管理”，因为：

- 字段语义清晰
- 传参方便
- 后期扩展容易
- 代码中可以统一写成 `cfg.xxx`

------

#### 一组布尔开关初始化

```matlab
cfg.use_fast_A = false;
cfg.use_fast_B = false;
cfg.dir_late = false;
cfg.dir_stagnation = false;
cfg.dir_elite_only = false;
cfg.dir_small_step = false;
```

这里全部先初始化为 `false`（逻辑假）。

这是一种非常典型、也非常稳的配置写法：

- 先默认**什么都不开**
- 再根据 `mode` 单独打开某些开关

这样做的好处是防止“脏状态”：

- 不会因为上一次模式遗留某个设置
- 不会出现没初始化就被读取的情况
- 每个模式的行为更可控、更可追踪

------

#### `cfg.stall_window = max(5, round(0.08 * max_iter));`

这行是在定义**停滞窗口（stagnation window，停滞判定窗口）**。

拆开看：

- `0.08 * max_iter`：取总迭代数的 8%
- `round(...)`：四舍五入为整数
- `max(5, ...)`：保证最小值不低于 5

意思就是：

> 停滞判定不能太短，否则太敏感；也不能完全固定，所以和总迭代数成比例。

例如：

- `max_iter = 100` 时，窗口约为 `8`
- `max_iter = 50` 时，窗口约为 `5`
- `max_iter = 500` 时，窗口约为 `40`

这个设计体现的是：

- 小实验时不让窗口太小
- 大实验时又能随着总时长增长而拉大判断尺度

------

#### `cfg.late_start = 0.60;`

这个变量表示：

> 晚期阶段从总体进度的 60% 开始。

通常主循环里会有：

```matlab
progress = t / Max_iteration;
```

因此：

- `progress = 0.2` 表示前期
- `progress = 0.6` 表示进入中后期
- `progress = 0.9` 表示后期

这类比例阈值写法比直接写“第 60 代”更通用，因为不同实验的迭代总数可能不同。

------

#### `switch char(mode)`

这里是 MATLAB 的 `switch-case`（多分支选择）语法。

关键点有两个：

1. `switch` 适合处理“一个变量对应多种模式”的情况
2. `char(mode)` 是为了把 `string`（字符串对象）或其他字符串形式统一转成字符数组，避免类型不一致导致比较失败

为什么不直接 `switch mode`？

因为 MATLAB 里有时：

- `mode` 可能是 `"dir_late"`（string）
- 也可能是 `'dir_late'`（char）

统一转 `char(mode)` 更稳。

------

#### `case 'baseline'`

这一分支故意什么都不做。

为什么？

因为前面已经把所有开关默认设成 `false` 了，所以 baseline 就是：

- 不开 A
- 不开 B
- 不开 directional 特殊策略

这是非常典型的“空分支作为基线”的写法。

------

#### `case 'fast_simple_a'`

```matlab
cfg.use_fast_A = true;
```

这表示：

- 打开 A 模块
- 其他模块不动，仍然保持 false

这就是单模块消融的核心思想：

> 每次只打开一个改进，看它单独会带来什么效果。

------

#### `case 'dir_small_step'`

```matlab
cfg.dir_small_step = true;
cfg.dir_late = true;
```

这里很关键。

它不是只开一个 `small_step`，还顺带开了 `dir_late`。

这说明这个模式的真实含义不是：

> 任何时候都小步长 directional

而是：

> 在 late stage（后期阶段）启用 directional，并且 directional 的步长被缩小。

也就是说，这是一个**组合语义**：

- `dir_late` 决定“什么时候触发”
- `dir_small_step` 决定“触发后怎么走”

这点在看结果时要特别注意，因为它不是纯单因子。

------

#### `case 'hybrid_b_dir_small'`

```matlab
cfg.use_fast_B = true;
cfg.dir_small_step = true;
cfg.dir_late = true;
```

这个模式一次开了三个标志：

- `use_fast_B`
- `dir_small_step`
- `dir_late`

表示这是一个真正的混合模式：

- B 模块负责后期收缩开发
- directional 模块负责方向引导
- late 开关负责避免过早扰动

------

#### `otherwise`

```matlab
error('Unsupported v3 ablation mode: %s', mode);
```

这行非常重要，它不是“可有可无的报错”。

它的作用是：

- 防止模式名拼错后悄悄跑成 baseline
- 防止新加模式但没写映射逻辑
- 保证实验可复现，因为错误会立即暴露

`%s` 是字符串占位符，`mode` 会被格式化填进去。

例如如果传入：

```matlab
mode = 'dir_smallstep'
```

少了下划线，就会抛出类似：

```matlab
Unsupported v3 ablation mode: dir_smallstep
```

------

### 2.3 这一段对主循环有什么意义

主循环通常不会写：

```matlab
if mode == 'fast_simple_a'
```

而是写成：

```matlab
if cfg.use_fast_A
```

这种写法更好，因为：

1. 主循环和模式名解耦
2. 同一个模块可以被多个模式共享
3. 组合模式更容易写
4. 后面做新版本时只改 `mode_config` 即可

也就是说：

- `mode` 是**实验层命名**
- `cfg.xxx` 是**代码层行为开关**

## 3. 各版本改进点 + 对应代码块

### 3.1 V3_BASELINE

入口代码：

```matlab
function [best_fitness, best_solution, Convergence_curve] = BBO_v3_baseline(N, Max_iteration, lb, ub, dim, fobj)
% V3 baseline wrapper for dual-objective ablation study.
    [best_fitness, best_solution, Convergence_curve] = BBO_improved_v3(N, Max_iteration, lb, ub, dim, fobj);
end
```

改进点：

- 无新增模块，作为消融锚点。

语法解析：

#### 这是一个标准包装器（wrapper，封装入口）

它本身几乎不做算法逻辑，只做一层转发：

- 输入照收
- 输出照返
- 中间调用真正实现函数

------

#### 输出列表

```matlab
[best_fitness, best_solution, Convergence_curve]
```

这是 MATLAB 的**多输出返回**语法。

表示函数结束后返回三个变量：

- `best_fitness`：当前运行找到的最优适应度
- `best_solution`：对应的最优解向量
- `Convergence_curve`：每轮或每阶段记录的收敛曲线

------

#### 输入参数含义

- `N`：种群规模（population size，个体数）
- `Max_iteration`：最大迭代次数
- `lb`：下界（lower bound）
- `ub`：上界（upper bound）
- `dim`：维度
- `fobj`：目标函数句柄（function handle，函数引用）

这里的 `fobj` 很关键。MATLAB 中常见写法是：

```matlab
fitness = fobj(x);
```

意味着把一个候选解 `x` 传入目标函数求值。

------

#### 主体调用

```matlab
[best_fitness, best_solution, Convergence_curve] = BBO_improved_v3(N, Max_iteration, lb, ub, dim, fobj);
```

这里是**把下层函数的返回值直接透传**回来。

也就是说：

- `BBO_v3_baseline` 不是重新实现了一遍算法
- 它只是给 benchmark 框架一个单独可调用的名字

这样做的好处是：

- baseline 和改进版可以在同一个算法列表中并列
- 不会因为入口不同影响实验管理
- 后期统计时名称更清晰

------

### 3.2 V3_FAST_SIMPLE_A

入口代码：

```matlab
function [best_fitness, best_solution, Convergence_curve] = BBO_v3_fast_simple_A(N, Max_iteration, lb, ub, dim, fobj)
% V3 fast-simple A: adaptive shrinkage + incumbent-centered late fine search.
    [best_fitness, best_solution, Convergence_curve] = BBO_improved_v3_ablation_core( ...
        N, Max_iteration, lb, ub, dim, fobj, 'fast_simple_a');
end
```

核心改进代码（A 模块）：

```matlab
if cfg.use_fast_A
    shrink = 1 - 0.50 * progress;
    trial = old_position + shrink .* (trial - old_position);

    if progress > 0.65 && rand < 0.35
        trial = trial + (0.08 + 0.18 * progress) .* (best_solution - trial);
    end

    if progress > 0.80
        fine_scale = 0.008 * (1 - progress + 0.1);
        trial = trial + fine_scale .* (ub - lb) .* randn(size(trial));
    end
end
```

改进含义：

- 中后期自适应收缩步长，减少无效大步。
- 晚期增强向当前最优靠拢并保留细粒度扰动。

语法解析：

#### 入口层的 `...`

```matlab
BBO_improved_v3_ablation_core( ...
    N, Max_iteration, lb, ub, dim, fobj, 'fast_simple_a');
```

`...` 是 MATLAB 的**续行符**，表示这一行代码还没结束，下一行接着算。

它的作用只是让代码排版更清晰，不改变语义。

------

#### `if cfg.use_fast_A`

表示：

> 只有当前模式打开了 A 模块，下面这段逻辑才执行。

也就是说这一块是可开关的，不会影响别的模式。

------

#### `shrink = 1 - 0.50 * progress;`

这是一个随进度线性变化的缩放系数。

当：

- `progress = 0` 时，`shrink = 1`
- `progress = 0.5` 时，`shrink = 0.75`
- `progress = 1` 时，`shrink = 0.5`

也就是说：

> 迭代越往后，允许的移动幅度越小。

这是一种典型的 exploitation（开发/局部利用）增强思路。

------

#### `trial = old_position + shrink .* (trial - old_position);`

这句是 A 模块最核心的一句。

先看数学意义：

- `trial - old_position`：原始候选步长向量
- `shrink .* (...)`：把这个步长缩短
- 再加回 `old_position`：得到收缩后的新位置

它不是简单把 `trial` 乘一个数，而是：

> 以原位置 `old_position` 为锚点，把候选点拉回去一点。

这在几何上相当于：

- 原来准备迈 1 步
- 现在只迈 `shrink` 倍的那一部分

------

#### 为什么用 `.*` 而不是 `*`

MATLAB 中：

- `*`：矩阵乘法
- `.*`：逐元素乘法（element-wise multiplication，按元素相乘）

这里 `shrink` 是标量，理论上 `*` 也能算，但写成 `.*` 更统一，表示这里采用的是逐元素缩放语义。

尤其当以后 `shrink` 扩展成按维度不同的向量时，`.*` 仍然成立。

------

#### 第一层晚期拉近

```matlab
if progress > 0.65 && rand < 0.35
    trial = trial + (0.08 + 0.18 * progress) .* (best_solution - trial);
end
```

这句含义是：

- 只有进度超过 65% 才考虑
- 而且只以 35% 的概率触发
- 一旦触发，就把 `trial` 朝 `best_solution` 再拉近一点

拆解：

##### `progress > 0.65`

表示这是**中后期逻辑**，不希望前期就过强地往最优个体收缩。

##### `rand < 0.35`

`rand` 生成 `[0,1)` 的均匀随机数。
因此这是一个**概率门控**。

这很重要，因为如果每次都强行向 best 靠拢，容易早熟（premature convergence，过早收敛）。

##### `(best_solution - trial)`

这是从 `trial` 指向当前最优解的方向向量。

##### `(0.08 + 0.18 * progress)`

这是拉近强度，随进度增大。

举例：

- `progress = 0.65` 时，系数约 `0.197`
- `progress = 0.90` 时，系数约 `0.242`

所以越后期，越敢往 best 附近压缩。

------

#### 晚期细粒度扰动

```matlab
if progress > 0.80
    fine_scale = 0.008 * (1 - progress + 0.1);
    trial = trial + fine_scale .* (ub - lb) .* randn(size(trial));
end
```

这部分的目的不是大范围跳，而是：

> 在 very late stage（很后期）做一个极小尺度的局部微调。

##### `randn(size(trial))`

`randn` 产生标准正态分布随机数。
`size(trial)` 保证生成和 `trial` 同维度的向量。

所以这一项本质是：

- 给每一维都加一个随机微扰
- 扰动均值为 0
- 有正有负

##### `(ub - lb)`

这是搜索区间宽度向量。乘上它的意义是：

> 让扰动尺度和问题本身的取值范围对齐。

否则如果不同问题的上下界差异很大，固定扰动会不合理。

##### `fine_scale = 0.008 * (1 - progress + 0.1)`

这个系数越到后期越小。

例如：

- `progress = 0.80` 时，约 `0.0024`
- `progress = 1.00` 时，约 `0.0008`

这说明它真的是**微扰**而不是大步搜索。

------

### 3.3 V3_FAST_SIMPLE_B

入口代码：

```matlab
function [best_fitness, best_solution, Convergence_curve] = BBO_v3_fast_simple_B(N, Max_iteration, lb, ub, dim, fobj)
% V3 fast-simple B: dimension-wise contraction with late intensified exploitation.
    [best_fitness, best_solution, Convergence_curve] = BBO_improved_v3_ablation_core( ...
        N, Max_iteration, lb, ub, dim, fobj, 'fast_simple_b');
end
```

核心改进代码（B 模块）：

```matlab
if cfg.use_fast_B
    contraction = 0.06 + 0.24 * progress;
    trial = trial + contraction .* (best_solution - trial);

    if progress > 0.70
        trial = 0.75 * trial + 0.25 * best_solution;
    end

    if progress > 0.85
        fine_scale = 0.005 * (1 - progress + 0.05);
        trial = best_solution + fine_scale .* (ub - lb) .* randn(size(trial));
    end
end
```

改进含义：

- 更强的收缩式 exploitation，后期在最优邻域做更细搜索。

语法解析：

#### `contraction = 0.06 + 0.24 * progress;`

这是一个逐渐增大的收缩强度：

- 初期约 `0.06`
- 后期可接近 `0.30`

比起 A 模块“先压缩原始步长”，B 模块更直接：

> 直接把 `trial` 往 `best_solution` 的方向拽。

------

#### `trial = trial + contraction .* (best_solution - trial);`

这句是典型的**线性插值/凸组合式靠近**。

从代数上可以改写为：

```matlab
trial = (1 - contraction) * trial + contraction * best_solution;
```

这说明：

- `contraction` 越大，结果越接近 `best_solution`
- `contraction` 越小，越保留原 trial

所以它本质是在做：

> 当前候选点与当前最优点之间的加权融合。

------

#### `if progress > 0.70`

```matlab
trial = 0.75 * trial + 0.25 * best_solution;
```

这句进一步收缩。

它和上一句的区别在于：

- 上一层是“随 progress 变化的基础收缩”
- 这一层是“到 70% 之后额外再压一次”

可以把它理解成二次收缩。

从形式上看，它也是凸组合（convex combination，加权平均）：

- 75% 保留当前候选
- 25% 引入最优个体信息

------

#### `if progress > 0.85`

```matlab
trial = best_solution + fine_scale .* (ub - lb) .* randn(size(trial));
```

这和 A 模块不一样。

A 的微扰是：

> 在当前 `trial` 附近微调

B 的微扰是：

> 直接以 `best_solution` 为中心，在它附近采样一个新点

所以 B 模块更强烈地表达了：

> 到非常后期时，把搜索中心切换到 best 邻域。

这在单峰或后期精修阶段通常更激进。

------

### 3.4 V3_DIR_LATE

入口代码：

```matlab
function [best_fitness, best_solution, Convergence_curve] = BBO_v3_dir_late(N, Max_iteration, lb, ub, dim, fobj)
% V3 directional module: active mainly in late stage.
    [best_fitness, best_solution, Convergence_curve] = BBO_improved_v3_ablation_core( ...
        N, Max_iteration, lb, ub, dim, fobj, 'dir_late');
end
```

核心触发代码（晚期触发）：

```matlab
trigger = false;
if cfg.dir_late && progress >= cfg.late_start
    trigger = true;
end
```

改进含义：

- directional 不在全程激活，避免前期破坏全局探索。

语法解析：

#### `trigger = false;`

这是一个很标准的触发标志初始化。

意思是：

- 默认不触发 directional
- 后面条件满足再改成 `true`

这种写法比直接在 if 内嵌套所有方向更新代码更清晰，因为：

- 触发逻辑和执行逻辑分离
- 后面可以叠加多个触发条件
- 调试时容易看状态

------

#### `if cfg.dir_late && progress >= cfg.late_start`

这里用到了逻辑与 `&&`。

MATLAB 里：

- `&&`：短路与（short-circuit AND）
- `&`：逐元素逻辑与

因为这里比较的是两个标量逻辑值，所以 `&&` 更合适。

它表示：

1. 模式上必须允许 `dir_late`
2. 当前进度必须到达晚期阈值

两个条件都满足，才触发。

------

#### 为什么要晚期触发

directional（方向引导）本质是在给搜索施加偏向。
前期更需要 exploration（全局探索），如果太早加方向约束，容易：

- 缩小搜索范围
- 减少多样性
- 提前向局部优区域塌缩

所以 `dir_late` 的设计理念是：

> 早期让原始 BBO 多搜，后期再做定向修正。

------

### 3.5 V3_DIR_STAGNATION

入口代码：

```matlab
function [best_fitness, best_solution, Convergence_curve] = BBO_v3_dir_stagnation(N, Max_iteration, lb, ub, dim, fobj)
% V3 directional module: activated by stagnation signal.
    [best_fitness, best_solution, Convergence_curve] = BBO_improved_v3_ablation_core( ...
        N, Max_iteration, lb, ub, dim, fobj, 'dir_stagnation');
end
```

核心触发代码（停滞触发）：

```matlab
if cfg.dir_stagnation && no_improve_count >= cfg.stall_window
    trigger = true;
end
```

改进含义：

- 在“连续无改进”时才触发 directional，针对卡住场景做定向救援。

语法解析：

#### `no_improve_count`

这个变量通常表示：

> 连续多少代没有刷新全局最好值。

常见更新逻辑会是：

- 如果这代有新 best，置 0
- 如果没有，`+1`

所以它是一个停滞程度指标。

------

#### `>= cfg.stall_window`

当连续无改进次数超过阈值，就认为：

> 算法可能陷入平台区或局部区域

这时触发 directional，相当于“给群体一次有方向的纠偏”。

与 `dir_late` 不同，这个模式不是按时间触发，而是按状态触发。

也就是：

- `dir_late`：看“到了什么时候”
- `dir_stagnation`：看“是不是卡住了”

------

### 3.6 V3_DIR_ELITE_ONLY

入口代码：

```matlab
function [best_fitness, best_solution, Convergence_curve] = BBO_v3_dir_elite_only(N, Max_iteration, lb, ub, dim, fobj)
% V3 directional module: elite-only directional vector construction.
    [best_fitness, best_solution, Convergence_curve] = BBO_improved_v3_ablation_core( ...
        N, Max_iteration, lb, ub, dim, fobj, 'dir_elite_only');
end
```

核心方向构造代码（精英约束）：

```matlab
if cfg.dir_elite_only
    elite_mid = elite_pool(max(2, round(size(elite_pool, 1) / 2)), :);
    candidate = best_solution + F .* (e1 - elite_mid) + tail .* randn(size(best_solution)) .* (e3 - best_solution);
else
    candidate = best_solution + F .* (e1 - e2) + tail .* randn(size(best_solution)) .* (e3 - best_solution);
end
```

改进含义：

- 用更保守的精英向量替代普通差分，强调稳定性。

语法解析：

#### `elite_pool(...)`

`elite_pool` 通常表示当前精英个体集合，比如适应度最好的前若干个体。

------

#### `size(elite_pool, 1)`

MATLAB 中：

- `size(A,1)`：行数
- `size(A,2)`：列数

如果 `elite_pool` 是“每一行一个个体”，那么行数就是精英数量。

------

#### `round(size(elite_pool, 1) / 2)`

表示取精英池中间位置的索引。

例如：

- 精英数为 6，则中间约为 3
- 精英数为 5，则中间约为 3

------

#### `max(2, round(...))`

这是一个保护操作，避免索引过小。

说明作者不想取第 1 个精英，而是至少取第 2 个位置。
原因通常是：

- 第 1 个往往就是 `best_solution`
- 若方向向量总是围绕 top-1，会过于激进
- 取一个中间精英，更稳一点

------

#### `elite_mid = elite_pool(..., :)`

这里的 `:` 表示取该行的**所有列**。

也就是：

> 取出一个完整的精英个体向量。

------

#### 方向候选公式

```matlab
candidate = best_solution + F .* (e1 - elite_mid) + tail .* randn(size(best_solution)) .* (e3 - best_solution);
```

分成三部分看：

1. `best_solution`
   作为方向更新的基准中心
2. `F .* (e1 - elite_mid)`
   一个差分方向项
   - `e1`：某个精英
   - `elite_mid`：中间精英
     这比 `e1 - e2` 更保守，因为两个向量都来自精英群内部，不容易过于发散
3. `tail .* randn(...) .* (e3 - best_solution)`
   一个带随机扰动的尾部修正项
   让搜索仍然有一点随机弹性，不是纯确定性地冲向某个方向

------

#### `else` 分支

```matlab
candidate = best_solution + F .* (e1 - e2) + tail .* randn(size(best_solution)) .* (e3 - best_solution);
```

这是普通差分式 directional。

和 elite-only 的区别是：

- 普通版差分：`e1 - e2`
- 精英约束版差分：`e1 - elite_mid`

后者的意图是减少方向构造中的波动幅度。

------

### 3.7 V3_DIR_SMALL_STEP

入口代码：

```matlab
function [best_fitness, best_solution, Convergence_curve] = BBO_v3_dir_small_step(N, Max_iteration, lb, ub, dim, fobj)
% V3 directional module: conservative small-step directional updates.
    [best_fitness, best_solution, Convergence_curve] = BBO_improved_v3_ablation_core( ...
    N, Max_iteration, lb, ub, dim, fobj, 'dir_small_step');
end
```

核心小步长代码：

```matlab
if cfg.dir_small_step
    F = 0.5 * F;
    tail = 0.35 * tail;
end
```

改进含义：

- 缩小 directional 振幅，降低越界和过冲风险。

语法解析：

#### `F = 0.5 * F;`

这是直接把差分主步长砍半。

如果原本公式里 `F` 控制主方向幅度，那么这句就是：

> 保留方向，但减少冲击力。

------

#### `tail = 0.35 * tail;`

`tail` 通常对应尾部扰动或随机扩散部分。
这句把它压到原来的 35%。

因此这两句联合效果是：

- 主方向变小
- 随机尾部也变小

所以 `dir_small_step` 不只是“小一点”，而是：

> 把 directional 的确定性部分和随机性部分都一起收缩。

这正是“保守 directional”的核心。

------

#### 为什么这段代码很关键

很多 directional 机制失败，不是方向不对，而是：

- 步长太大
- 叠加噪声太强
- 后期还在大幅摆动

这段代码等于是直接控制**更新振幅**，因此非常适合做：

- 稳定性增强
- 后期精细开发
- 降低复杂函数上偶发过冲

------

## 4. 稳定性保护代码（所有模式共享）

### 4.1 严格贪婪提交/回滚

```matlab
trial_fitness = fobj(trial);
if trial_fitness < old_fitness
    population(i, :) = trial;
    fitness(i) = trial_fitness;
else
    population(i, :) = old_position;
    fitness(i) = old_fitness;
end
```

意义：

- 新候选不优就回滚，避免无效试探污染种群。

语法解析：

#### `trial_fitness = fobj(trial);`

这是对新候选解 `trial` 做一次目标函数求值。

其中：

- `fobj`：函数句柄
- `trial`：当前候选向量

------

#### `if trial_fitness < old_fitness`

这里默认是**最小化问题**。

也就是说：

- 适应度越小越好
- 只有新值更小，才接受新解

如果是最大化问题，这里就要改方向。

------

#### `population(i, :) = trial;`

这句表示：

> 把种群中第 `i` 个个体整行替换成 trial。

MATLAB 中：

- `i`：第 `i` 行
- `:`：所有列

所以就是替换整条解向量。

------

#### 为什么 `else` 里还要显式写回旧值

```matlab
population(i, :) = old_position;
fitness(i) = old_fitness;
```

理论上如果之前没改动过，也许不写似乎也行。
但显式写回有两个好处：

1. 逻辑完整：接受/拒绝都写清楚
2. 防止前面某处已对 `population(i,:)` 做了临时覆盖

这叫**显式回滚（explicit rollback，明确回滚）**。

------

#### 这一段在算法性格上的意义

这是很强的 greedy selection（贪婪选择）：

- 不接受变差
- 每个个体本轮只保留更优结果

优点：

- 收敛更稳
- 不容易被坏候选污染

缺点：

- 多样性下降更快
- 某些复杂地形可能更保守

------

### 4.2 directional 仅替换最差个体

```matlab
[worst_fitness, worst_idx] = max(fitness);
if candidate_fitness < worst_fitness
    population(worst_idx, :) = candidate;
    fitness(worst_idx) = candidate_fitness;
end
```

意义：

- directional 注入有“下限保护”，优先修复尾部个体，风险更低。

语法解析：

#### `[worst_fitness, worst_idx] = max(fitness);`

MATLAB 中 `max` 不仅返回最大值，还能返回其索引。

所以这里同时得到：

- `worst_fitness`：当前最差适应度
- `worst_idx`：最差个体所在位置

因为这里是最小化问题，所以“最大 fitness”就是最差个体。

------

#### `if candidate_fitness < worst_fitness`

这说明 directional 生成的 `candidate` 不是随便塞进种群，而是必须满足：

> 至少比最差个体更好

换句话说，directional 注入的门槛是：

- 不要求优于 best
- 不要求优于当前个体
- 但至少要改善种群尾部

这是一个非常稳的注入策略，因为它减少了对优秀个体的扰动。

------

#### `population(worst_idx, :) = candidate;`

表示只替换最差个体，不动中间层和精英层。

所以 directional 在这里更像一种：

> 尾部修复器 / 群体下限抬升器

而不是全种群强行改写器。

------

#### 为什么这段保护很重要

如果 directional 直接替换任意个体，风险会很大：

- 可能破坏好个体
- 可能引起群体结构震荡
- 可能让改进模块侵入性过强

现在这种“只替换最差个体”的做法，相当于给 directional 上了一道保险。
