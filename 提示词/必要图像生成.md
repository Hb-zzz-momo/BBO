## 配置要求
请在 fill_default_config(cfg) 或等价配置函数中增加以下字段（若已存在则兼容保留）：

- cfg.enable_plots = true;              % 是否生成图像
- cfg.show_plots = false;               % 是否实时显示图像，默认 false
- cfg.save_plots = true;                % 是否保存图像，默认 true
- cfg.plot_formats = {'png'};           % 可扩展，至少支持 png
- cfg.plot_dpi = 200;                   % 导出分辨率
- cfg.plot_subdir = 'figures';          % 图像保存子目录

注意：
- show_plots=false 时，必须确保 figure 不会弹窗显示
- 即使 enable_plots=true，也只保存不显示
- 若 save_plots=false，则可以只构造图但不写文件（仍不显示）

## 输出目录要求
所有图片统一保存到：
result_dir / cfg.plot_subdir /

例如：
results/cec2017/exp_xxx/figures/
results/cec2022/exp_xxx/figures/

如果不存在则自动创建。

## 图像类型要求
先实现最通用、最稳妥的一类图，不要一次做太多复杂图：

### 第一优先级：收敛曲线图
要求：
1. 对每个函数生成一张 convergence curve（收敛曲线图）
2. 横轴：iteration 或 FE（根据现有数据结构选择最稳定的那个）
3. 纵轴：best-so-far fitness
4. 每条曲线对应一个算法
5. 如果有多次 runs，优先画 mean convergence curve（多次运行平均收敛曲线）
6. 图标题包含：
   - suite 名称
   - function id
   - dim
7. 文件命名规范清晰，例如：
   - conv_f01.png
   - conv_f02.png
   - ...
8. 不要求美化过度，但要求图例、坐标轴、标题完整
9. 若数据尺度差异过大，可考虑纵轴 log10 变换，但必须谨慎：
   - 若值可能<=0，则不要强行取 log
   - 可以优先保持原值
10. 若当前框架没有完整收敛历史数据，请补充“每次 run 的每代最优值记录”，但不要影响主流程。
