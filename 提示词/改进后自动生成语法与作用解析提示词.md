# 改进后自动生成“语法 + 作用解析文档”提示词包

用途：
每次你完成一次算法改进后，直接把本文件中的提示词发给 Codex，让它自动产出一份“改进点对应代码块 + MATLAB 语法解释 + 作用分析”文档。

---

## 1. 完整执行版（推荐）

你现在进入“改进后文档化执行模式”。

任务目标：
在本次代码改进完成后，自动生成一份可复用的技术文档，内容必须包含：
1) 改进点对应代码块
2) 每个代码块的 MATLAB 语法详解
3) 每个改进点的算法作用与预期影响
4) 与本轮实验配置/输出的对应关系

### 一、工作边界
1. 不重写算法，不再改动核心逻辑（除非发现阻断性错误）
2. 文档必须严格基于当前仓库真实代码，不允许编造
3. 仅解析本轮相关文件，不发散到无关模块
4. 若信息不足，明确写“缺失信息 + 影响”

### 二、输入来源（必须使用）
请从以下来源抽取事实：
1. 本轮新增/修改的算法文件
2. 本轮 runner/pipeline 文件
3. 本轮结果目录中的关键输出（summary、ranking、analysis）
4. 现有注释与函数名语义

### 三、文档必须覆盖的内容
#### 1）任务上下文
- 本轮改进目标
- 本轮比较版本
- 本轮函数集/协议摘要

#### 2）版本映射总览
- 每个版本相对 baseline 新增了什么
- 对应入口函数与 mode 映射

#### 3）改进点逐项解析（核心）
对每个改进点按固定结构输出：
- 改进点名称
- 对应代码块（原样摘录）
- 代码位置（文件 + 函数）
- 语法解析（逐行）
- 算法作用（为什么这样做）
- 风险与边界（何时可能副作用）

#### 4）关键 MATLAB 语法专项说明
至少解释：
- switch/case
- if-return 门控链
- 逻辑索引赋值
- 逐元素运算（.*, ./）
- 多返回值
- table 筛选与 sortrows
- fprintf 格式化写文件
- datestr + timestamp 路径

#### 5）与实验输出的对应关系
- 代码中的哪些字段会体现在 summary/ranking/analysis
- 哪些结论只能由 formal 给出，不能由 smoke 给出

#### 6）论文可复用内容
- 一段方法描述（严谨、不过度夸张）
- 一段实验解释（观察点与风险）

### 四、输出文件要求
请把文档保存到：
docs/02_method_design/improved/

文件名格式：
<本轮主题>_改进点与语法详解_<YYYYMMDD_HHMMSS>.md

例如：
V3_DIR_SMALL_STEP_reduced_改进点与语法详解_20260315_163000.md

### 五、格式要求
1. 使用清晰标题层级
2. 每个改进点必须带代码块
3. 代码块后必须紧跟“语法解析”和“作用解析”
4. 结尾附“速查表”（版本 -> 改进点 -> 代码位置 -> 主要影响）

### 六、质量自检（输出前必须执行）
1. 是否所有改进点都有对应代码块
2. 是否所有代码块都解释了语法而不只讲概念
3. 是否引用了本轮真实文件而非猜测
4. 是否区分 smoke 结论与 formal 结论
5. 是否包含风险项而非只写优点

完成后请输出：
1. 生成的文档路径
2. 覆盖的文件清单
3. 若有未覆盖项，说明原因

---

## 2. 轻量快速版（每轮可直接贴）

请基于本轮改进代码，自动生成一份“改进点 + 代码块 + MATLAB语法 + 作用分析”文档。

硬性要求：
1. 每个改进点必须给真实代码块
2. 每个代码块必须逐行语法解释
3. 每个改进点必须写“作用 + 风险”
4. 必须说明与本轮 smoke/formal 输出的对应关系
5. 不允许编造未实现机制

输出到：
docs/02_method_design/improved/
文件名：
<主题>_改进点与语法详解_<YYYYMMDD_HHMMSS>.md

文档结构：
- 背景与目标
- 版本映射
- 改进点逐项解析（代码块+语法+作用）
- 关键MATLAB语法速查
- 与实验结果对应关系
- 论文可复用描述
- 风险与后续建议

## 3. 占位符模板版（可参数化）

将以下占位符替换后发送：

你现在为本轮改进 [ROUND_TAG] 生成技术解析文档。

本轮主题：[TOPIC]
核心文件：
- [FILE_1]
- [FILE_2]
- [FILE_3]
结果目录：
- [RESULT_DIR]

请输出文档，必须包含：
1. 改进点 -> 对应代码块
2. 逐行 MATLAB 语法解析
3. 改进作用与边界风险
4. 与 smoke/formal 输出项的映射
5. 论文可复用方法段落

输出路径：
docs/02_method_design/improved/[TOPIC]_改进点与语法详解_[TIMESTAMP].md

---

## 4. 你每轮可复用的最短操作流

1. 完成代码改进
2. 跑完 smoke + formal_screen
3. 把“完整执行版”提示词发给 Codex
4. 检查文档是否包含：代码块、语法逐行解释、作用与风险
5. 将该文档纳入本轮实验归档
